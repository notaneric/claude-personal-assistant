# /write [topic or brief] — Humanized Content Pipeline

Produce written content that sounds like a real person wrote it — not an AI. Apply the humanizer + marketing-skills pipeline.

## Skills Activated
Primary: humanizer  
Secondary: marketing-skills, brand-guidelines  
Support: theme-factory (brand voice consistency)

## Steps

1. **Parse the brief:**
   - Content type: blog post, email, landing page, social, thread, case study, docs
   - Audience: technical, general, business, consumer
   - Tone: authoritative, conversational, urgent, educational
   - Length: short (<300w), medium (300-800w), long (800w+)
   - Goal: inform, persuade, convert, entertain

2. **Draft with intent:**
   - Lead with the most valuable thing (no warm-up paragraphs)
   - Make every sentence earn its place
   - Specific over vague: numbers, names, examples
   - No filler phrases: "In conclusion", "It's worth noting", "At the end of the day"

3. **Humanizer pass** (blader/humanizer):
   Remove AI tells:
   - Varied sentence length (not all 15-20 words)
   - Occasional sentence fragments for emphasis. Like this.
   - First-person voice where natural
   - Concrete sensory details over abstractions
   - Strategic imperfection (not clinical perfection)
   - Remove: "delve into", "it's important to note", "furthermore", "in today's world"
   - Remove: any opening that starts with "Certainly" or "Of course"

4. **Marketing check:**
   - CRO: does the CTA work?
   - Hook: does the opening hold attention?
   - Value prop: is it clear what the reader gets?

5. **Brand voice check:**
   - Consistent with the project's voice (reference any brand guidelines on file)
   - Direct, specific, no corporate filler

6. **Output:**
   - Final content ready to publish
   - Optional: 3 headline variants (A/B test candidates)
   - Optional: social media excerpt version

## Content Type Shortcuts
- `/write blog [topic]` — blog post (600-1200 words)
- `/write email [goal]` — cold/warm email, subject line included
- `/write landing [product]` — landing page copy (hero + features + CTA)
- `/write thread [topic]` — Twitter/X thread (10-15 tweets)
- `/write docs [feature]` — technical documentation, developer-friendly
